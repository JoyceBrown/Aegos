Aegos 2.9.12 到 5.0 开发路线与安全工程规划
生成目的：把成熟代理客户端、分流能力、IPv6/DNS 防泄漏、智能诊断、自愈、UI 精修、安全防护、架构清洗放到同一条前后对齐的路线里。
当前基准：Aegos 2.9.12


一、总原则

1. 先稳定底座，再新增复杂能力。
   连接、订阅、测速、系统代理、TUN、断网保护、落地 IP、诊断日志必须先稳定，否则不能急着做自定义规则和复杂分流。

2. 安全不是最后补丁，而是每个阶段的准入门槛。
   任何涉及订阅解析、配置生成、策略组、规则、系统代理、防火墙、TUN、PowerShell、文件读写、日志导出的功能，都必须有安全预检、失败回滚、日志脱敏、审计测试。

3. 规则能力必须分三步走。
   先只读分流，再规则底座，再普通用户分流助手。不能直接开放用户自定义规则，否则任何底层 bug 都会被放大。

4. IPv6 当前先关闭，后续做自动检测与防泄漏。
   后续目标是 IPv6 模式自动：检测本机 IPv6、当前节点 IPv6、IPv6 出口 IP、IPv6 泄漏，节点不支持时自动回退 IPv4 或阻断泄漏，并提示用户。

5. 所有用户可见功能必须能解释。
   连接失败、订阅失败、测速失败、规则不生效、DNS 泄漏、IPv6 泄漏、端口占用、权限不足、WebView2 缺失，都必须给出用户能看懂的原因和建议动作。

6. 测速永远不能自动连接。
   测速只更新延迟、推荐、低延迟列表、稳定性，不改变当前节点，不改变当前模式，不接管用户连接。

7. 推荐永远不等于当前连接。
   UI 必须区分当前节点和推荐节点，不能让用户误以为推荐已经连接。


二、安全威胁模型：Aegos 需要防什么

1. 恶意订阅攻击
   风险：
   - 恶意 YAML / URI 让解析器崩溃。
   - 恶意订阅构造不存在策略组、循环引用、非法协议，导致核心启动失败。
   - 恶意 remote provider / rule provider 诱导二次下载。
   - 订阅 URL token 泄露到日志或诊断报告。
   - 订阅内容插入异常路径、异常端口、异常命令文本。

   动作：
   - 订阅解析统一走安全解析器。
   - 订阅导入前做格式、协议、策略组、规则引用、端口、路径预检。
   - 订阅 token 在日志、诊断、导出报告中脱敏。
   - 限制或明确提示远程 provider 行为。
   - 异常订阅必须失败回滚，不能污染当前可用配置。
   - 给订阅解析器增加 fuzz / fixture 回归测试。

2. 本地控制接口攻击
   风险：
   - mihomo controller 端口被其他本地程序调用。
   - 控制端口暴露到局域网。
   - 缺少 secret 或鉴权策略，导致本机恶意程序改节点、关代理、读连接。

   动作：
   - controller 默认只绑定 127.0.0.1。
   - allow-lan 默认关闭。
   - controller secret 自动生成并保存。
   - 所有 controller 请求统一封装，禁止散落调用。
   - 诊断页检查 controller 绑定地址、端口、secret、allow-lan 状态。

3. 系统代理 / TUN / 防火墙误操作
   风险：
   - 开启失败后系统代理残留。
   - 断网保护规则残留导致断网。
   - TUN 半启动导致路由异常。
   - 管理员权限动作被普通 UI 误触发。

   动作：
   - 所有系统级变更必须事务化：预检、应用、验证、失败回滚。
   - 开启前保存系统代理快照，退出/失败时恢复。
   - 防火墙规则必须有 Aegos 前缀、可枚举、可清理。
   - TUN、断网保护必须检测管理员权限，不满足时给明确提示。
   - 增加“修复系统代理/清理防火墙规则/恢复网络”诊断动作。

4. 数据泄露
   风险：
   - 日志导出包含订阅 token、机场地址、真实 IP、节点密码。
   - 诊断报告包含敏感路径、订阅 URL、密钥。
   - 配置文件权限过宽，被其他用户读取。

   动作：
   - 日志、诊断、导出报告统一脱敏。
   - 订阅 URL token、password、uuid、private key、server、真实公网 IP 按级别脱敏。
   - 配置目录权限检查。
   - 后续使用 Windows DPAPI 或系统安全存储保护敏感信息。
   - 导出诊断包前显示“已脱敏”状态。

5. 命令注入 / 脚本注入
   风险：
   - PowerShell 参数拼接不安全。
   - 防火墙规则名、路径、端口、订阅名、节点名进入命令。

   动作：
   - 所有 PowerShell 输入统一转义。
   - 命令调用集中在少数安全封装函数。
   - 禁止把用户可控字符串直接拼到 shell 命令。
   - 审计脚本检查危险拼接模式。

6. 路径穿越 / 文件污染
   风险：
   - 订阅名、节点名、导出名影响文件路径。
   - 临时文件未清理。
   - 运行时配置覆盖用户文件。

   动作：
   - 所有文件名使用内部 id，不直接用用户输入。
   - 路径必须限制在 Aegos app data 目录。
   - 写文件走临时文件 + 原子替换。
   - 删除/移动前校验路径位于工作目录或 app data 内。

7. WebView / 前端攻击
   风险：
   - XSS 或 HTML 注入。
   - 外部内容进入 WebView。
   - Tauri IPC 命令权限过宽。

   动作：
   - 用户文本只作为 textContent 渲染，不写 innerHTML。
   - CSP 保持严格，不加载远程脚本。
   - Tauri capability 最小授权。
   - 新增命令必须列入 ACL 审查。
   - 前端 smoke 检查危险 innerHTML 和过宽权限。

8. 供应链与发布风险
   风险：
   - mihomo.exe 被替换。
   - 依赖包存在已知漏洞。
   - 安装包被篡改。
   - 自动更新被劫持。

   动作：
   - 记录并校验 bundled mihomo.exe hash。
   - npm/cargo 依赖审计。
   - 发布包 SHA-256 固化到 release notes。
   - 4.x 规划代码签名和安全更新机制。
   - 自动更新必须签名校验后才能启用。

四、2.9.13 - 2.9.19：当前功能补漏与安全热修

2.9.13：连接闭环复测
- TUN 关闭时系统代理连接闭环。
- 连接、断开、重连、核心待命状态一致。
- 失败后系统代理不残留。
- 安全动作：连接失败日志脱敏，系统代理快照恢复检查。

2.9.14：测速闭环复测
- 一键测速、批量测速、单节点测速绝不切换节点。
- 测速中切页面、切筛选、切节点结果不丢。
- 测速失败不能永远显示测速中。
- 安全动作：断网保护开启时测速的临时防火墙放行必须可清理。

2.9.15：订阅切换与更新复测
- 异常订阅导入、更新、切换失败回滚。
- 订阅切换后当前连接状态一致。
- 协议兼容继续查缺。
- 安全动作：订阅 URL、token、节点密码日志脱敏。

2.9.16：状态一致性复查
- 当前节点、推荐节点、落地 IP、局域网 IP、系统代理状态一致。
- 智能分流下落地 IP 查询仍代表当前节点出口。
- 安全动作：公网 IP、局域网 IP、IPv6 信息按日志级别脱敏。

2.9.17：断网保护复查
- 断网保护不影响测速、诊断、订阅更新。
- 开启失败自动回滚。
- 关闭后防火墙规则清理干净。
- 安全动作：防火墙规则命名、枚举、删除统一封装。

2.9.18：首次安装与环境检查
- WebView2 缺失提示。
- 权限不足提示。
- 端口占用提示和占用程序。
- 安全动作：默认 allow-lan 关闭，controller 只绑定 127.0.0.1。

2.9.19：UI 稳定收尾
- 导航不卡顿。
- 弹窗层级稳定。
- 不同窗口尺寸布局不跳。
- 安全动作：确认 UI 不使用危险 HTML 注入渲染用户输入。

二十、每个版本的安全准入清单

任何版本发布前必须检查：
1. 是否新增用户输入入口。
2. 是否新增文件读写。
3. 是否新增网络请求。
4. 是否新增系统级操作。
5. 是否新增 Tauri command。
6. 是否新增 PowerShell 或命令调用。
7. 是否新增配置生成或修改。
8. 是否会把敏感信息写入日志。
9. 是否失败后能回滚。
10. 是否有 audit/smoke/test 覆盖。

如果答案中有任意一项是“是”，必须补对应安全审查，不能只改功能。
