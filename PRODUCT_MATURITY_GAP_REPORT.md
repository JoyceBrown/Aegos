# Aegos 3.4.11 全软件真实完成度审计报告

本报告是 3.4.11 的主要产物。结论先写在前面：Aegos 已经有大量工程地基，但还不能把“审计通过”当成“产品成熟”。3.4.12 到 3.4.20 必须按真实用户任务补齐连接、测速、订阅、首页、分流、诊断日志、设置安全、安装和全局性能。

## 审计口径

本次不用单一“通过 / 不通过”评价页面，而是按三档判断。

- 工程完成：入口、后端命令、基础状态和审计存在。
- 产品可用：普通用户能完成任务，看得懂状态，失败知道下一步。
- 成熟可交付：异常可恢复，页面不卡不乱，安全和安装门禁通过，可给用户安装包测试。

## 总体判定

当前版本是 3.4.11 产品成熟度补课起点。

- 工程完成度：较高。连接、测速、订阅、分流、诊断、日志、设置、安全审计都已有基础。
- 产品可用度：中等。主流程能覆盖，但部分页面仍偏工程化，异常解释和任务闭环不够统一。
- 成熟可交付度：未达标。3.4.20 前不能进入 3.5.x。

阻止进入 3.5.x 的原因：

- 连接、测速、订阅、分流、诊断日志、设置安装仍需要按真实用户任务重新验收。
- 过去反复出现过测速误连接、诊断锁页、订阅切换串状态、落地 IP 过期、按钮卡死、布局跳动、日志未脱敏等问题，必须变成固定门禁。
- 真实用户任务 smoke 还没有覆盖完整路径：导入订阅、切换订阅、连接节点、测速、切节点、查看落地 IP、诊断问题、导出日志、调整设置、创建分流规则、撤销规则。

## 页面与能力审计

### 首页

工程完成：
- 有当前节点、延迟、稳定性、活跃连接、协议、DNS、系统代理、TUN、断网保护、落地 IP、局域网 IP、上下行、快捷操作和常用节点区域。
- 有页面缓存、节点行缓存、分批渲染和导航延迟加载。

产品可用：
- 首页已经能作为主控台雏形使用。
- 仍需复核最小高度、最大高度、窄宽度下是否有布局跳动。
- 系统代理、TUN、断网保护之间的关系仍要继续用普通用户语言压缩解释。

成熟可交付缺口：
- 首页需要在 3.4.15 统一状态来源，避免各区域重复推导造成显示不一致。
- 落地 IP 旧查询过期、节点切换后刷新、系统代理未开启提示必须继续回归。

### 节点

工程完成：
- 节点页已有筛选、搜索、批量测速、单节点测速、编辑、收藏列。
- 大列表有窗口化、缓存和限制渲染。

产品可用：
- 用户能找到节点并测速。
- 失败原因仍必须避免退化成“未测速”。
- 单节点测速不能卡行按钮或全页。

成熟可交付缺口：
- 3.4.13 必须统一首页和节点页测速状态源。
- TUIC、anytls、Hysteria2、VLESS、Trojan、SS 的失败分类和调度要继续复核。
- 直连、策略组、机场元数据伪节点不能进入普通测速。

### 连接

工程完成：
- 有连接、断开、重启内核、系统代理、TUN、断网保护、后台任务、连接失败分类。
- 有系统代理快照恢复和核心生命周期事务。

产品可用：
- 用户可以完成连接和断开。
- 仍需确保 TUN 关闭时系统代理路径可用，手动系统代理不会自动连接。

成熟可交付缺口：
- 3.4.12 必须把核心运行、核心待命、已接管、系统代理、TUN、断网保护、当前节点、落地 IP 收敛成一个连接闭环。
- 失败状态必须显示原因、影响、建议动作、诊断入口。

### 订阅

工程完成：
- 有导入、更新、切换、重命名、协议兼容、异常订阅诊断、切换预览和事务回滚基础。

产品可用：
- 用户能导入和切换订阅。
- 订阅切换浮层、下方节点同步、旧测速取消仍需继续专项回归。

成熟可交付缺口：
- 3.4.14 必须把订阅导入、更新、切换统一成可解释、可回滚、不中断当前可用配置的用户闭环。
- 订阅 URL、token、UUID、密码必须在所有日志和导出中脱敏。

### 分流

工程完成：
- 有只读快照、策略组、规则解析、草稿预览、冲突提示、地区/网站/应用草稿入口。

产品可用：
- 目前仍偏工程调试页，不能当作成熟分流功能。
- 普通用户能否独立完成新增、验证、应用、撤销规则，当前不能判定为达标。

成熟可交付缺口：
- 3.4.16 必须按 `ROUTING_PRODUCTIZATION_RECOVERY_PLAN.md` 做产品化。
- 分流页需要从“规则堆叠”改成“当前怎么走 / 我要新增 / 草稿与验证 / 高级查看 / 风险诊断”。

### 诊断

工程完成：
- 有诊断后台任务、严重程度、摘要、复制报告和诊断结果列表。

产品可用：
- 用户可以运行诊断并看到结果。
- 必须继续复测诊断中切页不被锁住。

成熟可交付缺口：
- 3.4.17 需要把诊断结果统一成原因、影响、建议动作。
- 诊断任务必须有超时、取消或过期保护。

### 日志

工程完成：
- 有用户、运行、核心、诊断、调试分类，支持导出。

产品可用：
- 用户可以筛选和导出日志。
- 导出后的脱敏仍需作为硬门禁。

成熟可交付缺口：
- 3.4.17 必须增加导出脱敏检查，覆盖订阅 URL、token、UUID、密码、本地路径、敏感 IP。
- 大日志量需要分页或窗口化复核。

### 设置

工程完成：
- 有系统代理、TUN、DNS、断网保护、IPv6、端口、日志级别、可靠性、手动节点等设置基础。

产品可用：
- 用户可以调整设置。
- 设置页仍偏工程化，危险操作和高级项需要分区。

成熟可交付缺口：
- 3.4.18 必须分普通、高级、危险操作。
- 断网保护、TUN、系统代理、DNS、IPv6 自动模式要统一文案和失败解释。

### 安装

工程完成：
- 有 NSIS 构建、WebView2 bootstrapper、安装包审计、安装回归清单。

产品可用：
- 3.4.11 不打安装包。
- 3.4.20 必须重新打包并给 SHA-256。

成熟可交付缺口：
- 3.4.18 和 3.4.20 必须复核干净 Windows 设备缺依赖、缺权限、端口占用、防火墙失败提示。

### 安全

工程完成：
- 有 controller 绑定、allow-lan 默认关闭、ACL、PowerShell 隐藏执行、日志脱敏、配置原子写入和路径限制等基础。

产品可用：
- 安全能力主要在后台，不应让普通用户承担理解成本。

成熟可交付缺口：
- 安全门禁必须在每个涉及系统代理、防火墙、配置写入、订阅、日志导出的版本继续跑。
- 3.4.20 前要做最后一次敏感信息扫描。

### 性能

工程完成：
- 已有导航延迟加载、页面缓存、后台任务、节点列表窗口化、快速导航 smoke。

产品可用：
- 大部分页面已经避免明显前台阻塞。
- 过去仍出现过诊断、日志、测速导致卡顿，必须继续当作阻塞级回归。

成熟可交付缺口：
- 3.4.19 必须做全局交互与性能验收。
- 测速、诊断、订阅更新、落地 IP 查询同时发生时，UI 仍必须可操作。

## 固定回归坑

这些问题以后不能靠记忆处理，必须进入固定检查项。

- 测速误连接。
- 诊断锁页。
- 订阅切换串状态。
- 落地 IP 过期。
- 按钮卡死。
- 布局跳动。
- 日志未脱敏。

## 3.4.12 到 3.4.20 执行顺序

- 3.4.12：先修核心连接闭环，因为代理软件连接不稳就没有产品价值。
- 3.4.13：再修节点与测速，因为测速误连接、卡顿、失败原因不清会直接破坏信任。
- 3.4.14：订阅产品化，确保机场用户入口稳定。
- 3.4.15：首页作为主控台，统一信息和常用操作。
- 3.4.16：分流页产品化，补齐自定义规则闭环。
- 3.4.17：诊断与日志产品化，让问题可解释、可导出、可脱敏。
- 3.4.18：设置、安全与安装产品化，压住系统级风险。
- 3.4.19：全局交互与性能验收，确认不会再动不动卡住。
- 3.4.20：全软件成熟度验收，跑真实用户任务 smoke，打安装包。

## 3.4.11 结论

3.4.11 不是功能完成版，而是纠偏审计版。它的作用是把 Aegos 从“工程门禁驱动”改成“真实用户任务驱动”。从此以后，进入 3.5.x 的前置条件不是版本号走到，而是 3.4.20 全软件产品成熟度验收通过。
