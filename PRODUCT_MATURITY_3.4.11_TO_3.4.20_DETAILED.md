# Aegos 3.4.11 到 3.4.20 细化执行计划

本文件是 `PRODUCT_MATURITY_RECOVERY_PLAN.md` 的执行细则。它把 3.4.11 到 3.4.20 拆成可执行、可验收、可追责的小版本。

## 总原则

- 3.4.11 到 3.4.20 是产品成熟度补课，不是继续堆新功能。
- 每个小版本必须解决一类真实用户任务，不能只加审计脚本、release note 或版本号。
- 小版本默认不打安装包；3.4.20 作为全软件成熟度检查点，必须打安装包。
- 每个版本都要有 release note、验证命令、git 提交。
- 每个版本完成前必须检查：测速不切节点、重任务不卡 UI、日志脱敏、失败可恢复、页面不乱。
- 如果发现主流程失败或用户无法完成核心任务，当前版本必须先修完，不能继续下一个小版本。

## 统一验收模板

每个版本都按下面格式收口：

- 用户任务：普通用户这一步能完成什么。
- 工程动作：前端、后端、状态模型、审计或测试做了什么。
- 产品动作：文案、布局、反馈、错误解释做了什么。
- 安全动作：脱敏、权限、路径、端口、防火墙、controller、配置写入做了什么。
- 性能动作：后台任务、缓存、虚拟化、取消、超时、分批渲染做了什么。
- 回归动作：哪些旧问题必须复测。
- 停止条件：发现什么问题必须暂停推进。

## 3.4.11：全软件真实完成度审计

### 用户任务

用户暂时不拿新安装包；本版本用于把 Aegos 当前所有半成品、隐藏风险、体验问题列清楚，防止继续盲目前进。

### 工程动作

- 建立 `PRODUCT_MATURITY_GAP_REPORT.md`。
- 建立 `tools/product-maturity-audit.js`。
- 审查首页、节点、连接、订阅、分流、诊断、日志、设置、安装包、安全、性能。
- 每个页面按三档标记：工程完成、产品可用、成熟可交付。
- 把已经踩过的坑列成固定检查项：测速误连接、诊断锁页、订阅切换串状态、落地 IP 过期、按钮卡死、布局跳动、日志未脱敏。

### 产品动作

- 为每个页面写用户任务清单。
- 为每个页面写普通用户视角的“看得懂 / 点得明白 / 出错知道怎么办”评分。
- 把调试型信息、工程术语、重复信息列为体验债务。

### 安全动作

- 检查日志、诊断导出、错误提示是否可能暴露订阅 token、UUID、密码、本地路径、真实敏感 IP。
- 检查 controller 绑定、allow-lan、端口、ACL、PowerShell 命令拼接风险。

### 性能动作

- 列出所有可能阻塞 UI 的路径：测速、诊断、订阅更新、核心启动、落地 IP、日志加载、规则渲染。
- 标记是否有取消、超时、后台任务、分批渲染。

### 产物

- `PRODUCT_MATURITY_GAP_REPORT.md`
- `tools/product-maturity-audit.js`
- `RELEASE_3.4.11.md`

### 验收

- 缺口报告必须包含阻塞项、重要项、体验项、长期项。
- 必须明确哪些问题阻止进入 3.5.x。
- 不能把“暂无发现”当结论，除非有对应检查证据。

### 停止条件

- 如果发现连接、测速、订阅、安装任一主流程不稳定，后续版本优先修主流程，不继续做美化。

## 3.4.12：核心连接闭环产品化

### 用户任务

用户能理解并完成：连接、断开、切换节点后继续使用、失败后知道原因。

### 工程动作

- 统一连接状态模型：核心运行、核心待命、已接管、系统代理、TUN、断网保护、当前节点、落地 IP。
- 检查连接按钮乐观 UI 与后端真实状态同步。
- TUN 关闭时系统代理路径必须可连接。
- 手动开系统代理不能自动连接。
- 断开连接必须恢复系统代理快照，不能残留。
- 落地 IP 查询必须有任务 id，切换节点后旧查询过期。
- 连接失败必须进入统一错误分类，不允许只显示原始异常。

### 产品动作

- 连接区只显示用户能理解的状态：未连接、连接中、已连接、断开中、失败。
- 失败提示必须包含：原因、影响、建议动作、诊断入口。
- 系统代理、TUN、断网保护的关系用短文案解释，不让用户猜。

### 安全动作

- 系统代理修改事务化，失败回滚。
- 防火墙或断网保护失败不得污染规则。
- 日志中不记录敏感节点凭据。

### 性能动作

- 连接、断开、落地 IP 查询全部后台化。
- 导航和普通按钮不能因为连接流程被锁。

### 回归动作

- 复测：TUN 关闭连接失败问题。
- 复测：手动开系统代理自动连接问题。
- 复测：切节点后落地 IP 仍显示旧查询问题。
- 复测：连接按钮点下后状态不立即变化问题。

### 产物

- 连接闭环修复代码。
- 连接状态审计脚本或扩展现有 stability/release audit。
- `RELEASE_3.4.12.md`

### 验收

- 连接、断开、失败、回滚、重连、切节点后刷新落地 IP 全部通过。
- 失败不会让 UI 卡在连接中或核心待命误状态。

## 3.4.13：节点与测速产品化

### 用户任务

用户能测速、看结果、理解失败原因，并且测速不会替用户连接节点。

### 工程动作

- 首页和节点页使用同一测速状态源。
- 单节点、一键、批量测速统一进入同一个任务模型。
- 测速任务有批次 id；订阅切换后旧批次自动失效。
- 失败结果必须结构化：超时、DNS、TLS、认证、协议不支持、目标不可达、被阻断、核心未就绪。
- 协议调度区分 SS、Trojan、VLESS、TUIC、anytls、Hysteria2。
- 直连、策略组、机场流量信息、过期信息等伪节点不进入普通测速。
- 大量失败节点不能让按钮和页面卡住。

### 产品动作

- 节点行显示：延迟或失败原因，不再只显示“未测速”。
- 单节点按钮显示局部状态，不锁全页。
- 首页快捷测速和节点页批量测速结果即时同步。
- 失败原因用短标签，详情放 tooltip 或诊断入口。

### 安全动作

- 测速目标不泄露订阅 token。
- 断网保护开启时测速临时放行必须可清理。
- 测速不能切换节点、模式、系统代理或接管流量。

### 性能动作

- 快速首轮 + 后台补测继续保留，但要确保结果流式返回。
- 对慢协议设置独立超时和并发上限。
- 渲染分批刷新，避免每个节点结果都触发全表重绘。

### 回归动作

- 复测：测速后自动连接。
- 复测：TUIC/anytls 大范围测不通。
- 复测：FlClash 虚拟网卡开启时 Aegos 测速异常。
- 复测：单节点测速卡按钮。
- 复测：测速中切筛选结果丢失。

### 产物

- 测速任务模型修复。
- 测速结果同步修复。
- 测速失败原因 UI。
- `RELEASE_3.4.13.md`

### 验收

- 测速不会连接或切换节点。
- 失败节点大面积存在时 UI 仍可操作。
- 首页和节点页结果一致。

## 3.4.14：订阅产品化

### 用户任务

用户能导入、更新、切换、重命名订阅；失败时知道是链接、网络、协议、格式还是机场问题。

### 工程动作

- 订阅导入、更新、切换统一事务模型。
- 切换订阅立即使用本地缓存预览节点，再后台验证。
- 切换订阅时取消旧测速、旧刷新、旧落地 IP 查询。
- 异常订阅不影响当前可用订阅。
- 支持主流订阅协议和 URI 格式，未知字段保留但不污染运行配置。
- 订阅重命名持久化。

### 产品动作

- 订阅卡片显示：名称、节点数、上次更新、当前使用、异常状态。
- 切换订阅浮层稳定，不闪跳、不被遮盖。
- 导入失败显示分类原因和建议。

### 安全动作

- 订阅 URL、token、节点密码、UUID 全部脱敏。
- 订阅配置写入限制在 app data。
- 更新失败不能覆盖旧可用配置。

### 性能动作

- 更新订阅后台化，不锁导航。
- 大订阅解析不阻塞首屏。
- 切换订阅不等待远端网络才更新本地显示。

### 回归动作

- 复测：异常订阅切换失败。
- 复测：切换订阅下方节点同步慢。
- 复测：切换订阅浮层闪跳。
- 复测：切换订阅旧测速污染新订阅。

### 产物

- 订阅事务和取消逻辑。
- 订阅错误分类。
- `RELEASE_3.4.14.md`

### 验收

- 多订阅连续切换不卡、不串、不闪。
- 异常订阅失败可解释且可回滚。

## 3.4.15：首页与常用操作产品化

### 用户任务

用户打开首页就知道：现在是否可用、走哪个节点、落地 IP 是什么、系统代理是否开启、下一步点什么。

### 工程动作

- 首页状态数据统一来源，不使用重复推导。
- 当前节点、延迟、稳定性、活跃连接、上次测速、落地 IP、局域网 IP、上下行统一刷新节奏。
- 常用地区默认香港，常用节点、收藏节点、固定节点逻辑不互相覆盖。
- 收藏和固定节点状态持久化策略明确。
- 快捷操作只保留高频且安全的动作。

### 产品动作

- 首页信息分层：连接状态优先，节点信息第二，快捷操作第三。
- 系统代理未开启必须明显提示。
- 无价值或重复信息删除。
- 最小高度、最大高度、窄宽度下不跳动。

### 安全动作

- 落地 IP、局域网 IP 按日志级别脱敏。
- 快捷操作不能触发危险动作无确认。

### 性能动作

- 首页列表行缓存和分批渲染。
- 后台刷新不能挤压点击反馈。

### 回归动作

- 复测：首页导航卡顿。
- 复测：常用地区/常用节点切换后延迟显示异常。
- 复测：不同窗口高度布局跳动。
- 复测：落地 IP 查询卡住。

### 产物

- 首页状态模型整理。
- 首页布局和快捷操作收口。
- `RELEASE_3.4.15.md`

### 验收

- 首页可以作为普通用户主控台使用。
- 快速切换筛选和节点不会乱序或跳动。

## 3.4.16：分流页产品化

### 用户任务

用户能新增一条分流规则，验证它，确认应用它，看到是否生效，并能撤销。

### 工程动作

- 分流页使用任务流状态：只读快照、草稿、预检、待确认、应用中、已应用、可撤销。
- 分流规则写入必须经过预检、原子写入、热重载、验证、回滚。
- 草稿支持网站、应用、地区、流媒体、AI 服务、游戏平台。
- 规则冲突、顺序覆盖、目标缺失、自动策略组覆盖必须结构化返回。

### 产品动作

- 页面重排为：当前怎么走、我要新增、草稿与验证、高级查看、风险诊断。
- 默认隐藏大规则表和策略组表，放到高级查看。
- 草稿卡显示来源、目的、影响、冲突、状态、下一步。
- 不用 YAML 作为主说明。

### 安全动作

- 用户输入只用 textContent 渲染。
- 写配置路径限制在 app data。
- 应用失败回滚，不能污染当前配置。

### 性能动作

- 大规则列表虚拟化或分页。
- 分流预检后台化，导航不锁。

### 回归动作

- 复测：分流页布局像调试页。
- 复测：分流页不清楚草稿是否生效。
- 复测：规则写入前缺少预检。

### 产物

- 分流页产品化 UI。
- 分流写入预检和回滚链路。
- 分流专用 smoke。
- `RELEASE_3.4.16.md`

### 验收

- 普通用户能完成新增、验证、应用、撤销一条规则。

## 3.4.17：诊断与日志产品化

### 用户任务

用户遇到问题时，能运行诊断、看懂结果、导出日志给开发者，同时不泄露敏感信息。

### 工程动作

- 诊断任务后台化，诊断中可切页。
- 诊断结果分严重、警告、信息。
- 日志分用户日志、调试日志、核心日志、诊断报告。
- 日志导出统一脱敏。

### 产品动作

- 诊断按钮显示诊断中，但不锁页面。
- 诊断结果显示原因、影响、建议动作。
- 日志页筛选简单清楚。

### 安全动作

- 导出前脱敏订阅 URL、token、UUID、密码、本地路径、敏感 IP。
- 核心日志中的节点名和连接目标按规则脱敏。

### 性能动作

- 日志分页或窗口化。
- 诊断任务有超时和取消。

### 回归动作

- 复测：诊断中不能切换导航。
- 复测：日志不能导出。
- 复测：诊断/日志页面容易卡。

### 产物

- 诊断日志任务模型和 UI 收口。
- 日志导出脱敏测试。
- `RELEASE_3.4.17.md`

### 验收

- 诊断中切页正常。
- 导出日志不含敏感字段。

## 3.4.18：设置、安全与安装产品化

### 用户任务

用户能理解设置项作用；安装到其他 Windows 设备时能运行，缺权限或依赖时知道怎么处理。

### 工程动作

- 设置页分普通、高级、危险操作。
- 断网保护、TUN、系统代理、DNS、IPv6 自动模式状态和说明统一。
- 端口冲突检测显示占用程序。
- WebView2、管理员权限、防火墙权限检查清晰。
- controller 绑定 127.0.0.1，默认 allow-lan 关闭。
- 安装包依赖和回滚检查进入审计。

### 产品动作

- 设置页去掉用户看不懂或没必要的工程细节。
- 危险操作有确认文案。
- 安装错误提示用普通话解释，不显示乱码。

### 安全动作

- ACL 最小化。
- 防火墙规则命名、枚举、删除统一封装。
- PowerShell 命令避免危险拼接。

### 性能动作

- 设置保存后台化，失败回滚。
- 安装审计不影响运行时性能。

### 回归动作

- 复测：断网保护功能异常。
- 复测：权限不足乱码。
- 复测：窗口 ACL 报错。
- 复测：端口冲突影响 Codex/FlClash。

### 产物

- 设置安全整理。
- 安装候选审计增强。
- `RELEASE_3.4.18.md`

### 验收

- 设置项用户能理解。
- 安装包候选安全门禁通过。

## 3.4.19：全局交互与性能验收

### 用户任务

用户快速点击、切页、测速、更新订阅、运行诊断时，软件仍像成熟桌面应用一样响应。

### 工程动作

- 审查所有主页面是否有同步重任务。
- 所有主列表：节点、连接、规则、日志、订阅，采用缓存、分页、窗口化或分批渲染。
- 统一 pending、busy、rollback、optimistic UI 策略。
- 后台任务优先级：用户点击高于后台刷新。
- 取消过期任务，防止旧结果覆盖新页面。

### 产品动作

- 按钮 hover、active、pending 不改变布局尺寸。
- 页面切换先确认，再异步刷新数据。
- 加载状态局部化，不遮全页。

### 安全动作

- 性能日志不记录敏感原文。
- 任务取消不会留下系统代理、防火墙、核心进程脏状态。

### 回归动作

- 复测：首页导航偶发卡顿。
- 复测：诊断和日志切换卡顿。
- 复测：测速失败节点拖死 UI。
- 复测：窗口尺寸变化布局错位。

### 产物

- 全局交互状态统一。
- 性能 smoke 扩展。
- `RELEASE_3.4.19.md`

### 验收

- 快速切换所有主页面无明显卡顿。
- 后台测速、诊断、订阅更新同时存在时 UI 仍可用。

## 3.4.20：全软件产品成熟度验收

### 用户任务

用户拿到安装包后，能完整测试 Aegos 作为代理客户端的核心能力。

### 工程动作

- 跑全套自动审计：release、安全、测速、响应性、稳定性、安装包、脱敏、回滚、产品成熟度。
- 跑真实任务 smoke：导入订阅、切换订阅、连接、断开、测速、切节点、查落地 IP、诊断、导出日志、改设置、创建分流规则。
- 记录已知风险和未完成项。

### 产品动作

- Release note 用普通话说明本版能测试什么、仍有哪些风险。
- 安装包版本、侧边栏版本、Cargo/Tauri/package 版本一致。

### 安全动作

- 安装包前最后一次敏感信息扫描。
