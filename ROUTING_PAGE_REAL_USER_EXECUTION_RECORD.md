# Aegos 分流页真实用户执行记录

本记录按 `ROUTING_PAGE_REAL_USER_ACCEPTANCE_STANDARD.md` 执行。版本号保持 `3.4.20`，本轮不推进版本号、不打包安装包、不用旧计划替代真实验收。

## 1. 本轮基线

结论：旧的 `ROUTING_PRODUCTIZATION_RECOVERY_PLAN.md` 已删除，`PRODUCT_MATURITY_RECOVERY_PLAN.md` 和 `PRODUCT_MATURITY_GAP_REPORT.md` 已改为引用新的硬标准。

本轮开始时分流页判定为未通过真实用户验收，原因如下：

- 后端快照不可用时，分流助手不渲染，普通用户只看到“分流数据不可用”。
- 设置页的“恢复策略 / 接管时记录”卡片误插入分流页摘要区，属于旧补丁串页。
- 1366x768 下草稿操作区被面板边界压住，视觉上像半截按钮。
- 高级信息区被 `details` 父容器裁成 1px，文本存在但用户点不到。
- 1120x720 最小窗口下触发了单列分流布局，导致核心按钮需要滚动才看完整。
- 旧审计脚本仍以早期只读分流阶段为准，不能证明当前真实产品路径完成。

## 2. 本轮修复

### 2.1 主路径先渲染

已修复：后端 `routing_snapshot` 失败时仍先渲染“创建分流规则 / 草稿与验证 / 高级信息”主路径。错误只显示在高级数据区，不再拿掉用户主任务。

验证结果：静态浏览器中 Tauri bridge 不可用时，分流助手仍显示，网站 / 应用 / 场景入口可用。

### 2.2 串页补丁修复

已修复：`ensureTakeoverControls()` 从全局 `.settings-summary-grid` 改为只作用于 `[data-page-panel="settings"] .settings-summary-grid`。

验证结果：分流页摘要区只剩 4 项：

- 当前模式
- 策略组
- 用户规则
- 系统规则

### 2.3 布局结构修复

已修复：

- 分流页网格从 `auto auto auto minmax(0, 1fr)` 改为 `auto auto auto auto`。
- 分流页自身启用 `overflow-y: auto` 和 `scrollbar-gutter: stable`。
- 高级信息区闭合时固定摘要高度，展开时按内容撑开。
- 低高度窗口下压缩分流页内部间距。
- 分流页单列断点从 `1120px` 下调到 `980px`，避免最小宽度误进单列。

验证结果：

- 1120x720：核心操作按钮完整露出，无横向溢出。
- 1366x768：核心操作按钮完整露出，无横向溢出。
- 1440x900：无横向溢出，主路径完整。
- 1920x1080：无横向溢出，主路径完整。

### 2.4 高级信息交互修复

已修复：

- 高级信息区闭合标题可见。
- 鼠标滚动到高级信息区后可点击展开。
- 去除浏览器默认橙色焦点框，改为 Aegos 风格的柔和焦点状态。

验证结果：高级信息区可展开，策略组和规则明细不再压过主任务。

### 2.5 错误原因分类修复

Rust 单元测试发现：`connect: network unreachable` 被分类为 `unreachable`，用户看到的原因会过泛。

已修复：带有 `connect` / `dial` 上下文的不可达错误归类为 `node-connect`，让用户看到“节点不通”这类更准确提示。

验证结果：`cargo test --manifest-path src-tauri/Cargo.toml` 通过，49 项测试全绿。

## 5. 当前结论

分流页当前达到“候选通过”：真实浏览器路径、布局、错误输入、草稿验证、高级区、响应性和工程审计已通过。

还不能声明“产品通过”的唯一原因：本轮没有在真实 Tauri 安装包环境中对“应用草稿 -> 写入配置 -> 热重载 -> 撤销恢复”做人工闭环复验。该项已有后端审计和单元测试证据，但最终产品通过前仍必须在真实运行环境复验。

禁止把旧的只读分流审计结论当成最终产品完成证据。

## 6. 本轮补充验收：摘要可查看、目标线路选择、草稿详情

本轮继续保持版本号 `3.4.20`，未推进版本号，未声明产品最终通过。

### 6.1 用户提出的问题

- 顶部“当前模式 / 策略组 / 用户规则 / 系统规则”不能只显示数字，必须能点开查看。
- “系统规则”必须解释清楚：它主要来自订阅配置和规则集，数量多是正常现象，普通用户通常不需要逐条改。
- “简单 / 高级”切换对普通用户没有意义，容易让人困惑。
- 网站、应用规则如果选择“走代理”，必须能选择使用哪个策略组、当前节点、收藏节点或固定节点；选择“直连 / 拒绝”时不显示无意义的线路选择。
- 更具体的网站 / 应用规则必须明确高于场景规则，避免用户担心冲突。
- 页面必须继续遵守草稿优先、验证不改配置、应用可撤销、不锁死界面、不引入危险 HTML 渲染的标准。

### 6.2 已修复内容

- 摘要卡变为可点击卡片，并支持键盘 Enter / Space 激活。
- 新增摘要详情区：
  - 当前模式：说明智能分流、全局代理、直连模式的含义。
  - 策略组：说明策略组是节点集合或自动选择器。
  - 用户规则：说明用户规则是更具体规则，默认高于场景规则。
  - 系统规则：按类型归纳订阅/规则集，不把大量底层规则直接压给普通用户。
- 删除全局“简单 / 高级”切换。
- 改为每条草稿自己的“详情 / 收起”，详情显示：
  - 底层规则。
  - 优先级。
  - 影响范围。
- 网站和应用规则新增“使用线路”下拉框，走代理时显示，直连/拒绝时隐藏。
- 下拉框选项统一来自只读分流快照：策略组、当前节点、收藏/固定节点、节点列表；静态环境无快照时保底显示“主要代理组”。
- 草稿详情长规则允许换行，避免用户看不到完整规则。
- 分流页 1180px 以下布局改为更稳的两列摘要、单列编辑器、收窄草稿操作列。
- 高级信息展开文案修正为“收起”，避免旧乱码露出。
- 为关键下拉框补充 `aria-label`，让控件语义和自动化验收更稳定。

## 7. 本轮补充验收：用户规则优先、应用状态、配置规则明细

本轮继续保持版本号 `3.4.20`，未推进版本号，未生成安装包。

### 7.1 冲突策略结论

采用面向用户的规则表达：

- 用户规则优先。
- 具体网站 / 应用规则优先于场景规则。
- 场景规则优先于订阅兜底规则。
- 如果出现“相同条件、不同目标”，不静默覆盖，必须提示用户确认。

这样比单纯说“用户规则优先”更准确：普通用户能理解，同时也保留了专业分流里的“更具体规则优先”原则。

### 7.2 截图区域处理结论

截图中的表格是“配置规则明细”，主要展示订阅配置和 Aegos 生成的底层规则。它不是普通用户的主任务区。

已调整：

- 高级区标题改为“配置规则明细”。
- 增加说明：这里是订阅和 Aegos 生成的底层规则，普通用户通常不需要逐条查看。
- 增加解释卡：主界面优先看用户规则和草稿；本区主要用于排查。
- 状态列不再显示无意义的“可查看”，改为“用户规则 / 配置规则 / 目标缺失 / 顺序风险”等更有意义的状态。

### 7.3 应用闭环可见反馈

已调整：

- 应用成功后显示最近应用状态：订阅、规则数量、是否可撤销。
- 撤销成功后显示已恢复到原配置。
- 应用失败或撤销失败时，分流页内保留失败状态卡，不只依赖临时 toast。
- 失败文案显示“原因”和下一步：查看日志、重试、撤销最近应用。
- 后端 `routing_snapshot` 返回 `lastApply` 和 `userRuleCount`。
- 后端根据最近一次 Aegos 应用元数据给当前规则打 `source=user`，前端不再猜测哪些订阅规则是用户规则。
